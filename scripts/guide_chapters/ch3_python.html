3.1 Why Python?

Python is known for its incredibly clean and readable syntax. Unlike languages like Java or C++ that require massive amounts of "boilerplate" code just to print a simple word, Python gets straight to the point. In SmartBank, Python is the engine of your backend. It handles security, calculates balances, and securely stores data in the SQL database.

3.2 Core Data Structures

This is a guaranteed viva question. You must know the difference between the four main data structures in Python: Lists, Tuples, Dictionaries, and Sets.

Mutability (The Most Important Concept)

Mutable means the data can be changed after it is created in memory. If you add an item to a list, it stays in the exact same memory location, it just gets bigger.

Immutable means the data cannot be changed. If you try to change a string in Python, the computer actually destroys the original string in memory and creates a brand new one. Strings, numbers, and Tuples are immutable.

1. Lists []

Lists are ordered, mutable, and allow duplicate values. Think of a shopping list. You can add items, remove items, and the order in which you wrote them down is preserved.

accounts = ["Savings", "Current", "Loan"]
accounts.append("Credit") # This works perfectly because lists are Mutable.

2. Tuples ()

Tuples are ordered and allow duplicates, but they are Immutable. Once you create a tuple, you can never add or remove items from it.

  • Why use them? Because they can't change, Python can process them much faster than lists. They also use less memory. You use tuples for data that should never be altered, like geographic coordinates (latitude, longitude) or RGB color codes.

3. Dictionaries {}

Dictionaries are unordered collections of Key-Value pairs. They are mutable. Think of an actual dictionary: you look up a word (the Key) to find the definition (the Value).

Dictionaries use Hash Tables under the hood. This means looking up a value takes almost zero time, even if the dictionary contains a million items. This is why we use dictionaries constantly in backend development.

user = {
    "id": 101,
    "name": "Salman",
    "balance": 50000
}
print(user["name"]) # Instantly prints "Salman"
Viva Tip: An examiner might ask, "Can a list be a key in a dictionary?" The answer is NO. Dictionary keys must be Immutable (like strings or tuples) so that their internal hash value never changes.

4. Sets {}

Sets are unordered, mutable, but they do not allow duplicates. If you have a list of a thousand transaction IDs and you want to instantly remove all the duplicates, you simply convert the list into a Set.

3.3 Memory Management and Garbage Collection

In languages like C, developers have to manually allocate and free memory from the computer's RAM. If they forget, it causes a "memory leak" and the computer eventually crashes. Python handles this automatically.

Python uses Reference Counting. Imagine a Python object in memory is a balloon. Every variable that points to that object is a string holding the balloon. If three variables point to it, the reference count is 3. If you delete those variables, the strings are let go. When the reference count drops to 0, Python instantly pops the balloon (frees the memory).

Python also runs a background Garbage Collector. Sometimes, Object A points to Object B, and Object B points to Object A, creating an isolated circle. Even if your main program isn't using them, their reference count isn't zero. The Garbage Collector's job is to detect these isolated circles and destroy them.

3.4 Object-Oriented Programming (OOP)

OOP is a way of organizing code so that it maps to real-world concepts. Instead of writing random functions, you create "Classes" and "Objects".

  • Class: The blueprint. Imagine architectural blueprints for a house. You can't live in a blueprint. It just defines what the house should look like (doors, windows, paint color).
  • Object (Instance): The actual house built from the blueprint. You can build 100 different houses from the same blueprint. In code, user1 and user2 are two different objects created from the User class.

The __init__ Method and self

In Python, the __init__ method is the constructor. It is a special function that runs automatically the moment you create an object. It "initializes" the object's properties.

The self keyword is highly confusing to beginners. It simply refers to the specific object being created. If you have a blueprint instruction that says "paint the door red", you need to know which house's door to paint. self.door = "red" means "paint my door red".

class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self.balance = balance

account1 = BankAccount("Salman", 1000) # __init__ runs automatically here!

3.5 Decorators (The '@' Symbol)

Since your project uses Flask, you use Decorators constantly (like @app.route). An examiner will definitely ask about this.

In Python, functions are "first-class citizens." This means a function is treated just like any other variable. You can pass a function into another function as an argument.

A Decorator is simply a function that takes another function, wraps it in some extra code, and returns the modified function. It allows you to "decorate" or modify a function without changing its internal code.

Example: In SmartBank, you might have a function called view_dashboard(). But you don't want anyone to see it unless they are logged in. You could write a massive if-statement inside the function. Or, you could write a decorator called @login_required. You place the decorator directly above the function. Before Python executes the dashboard logic, it runs the decorator's security check first.

3.6 Error Handling (Try / Except)

When bad things happen in code (like trying to divide by zero, or a database connection dying), Python throws an "Exception" and completely crashes the program. In a web server, a crash means the website goes offline for everyone.

To prevent this, we use Error Handling.

  • try: Python attempts to run this block of code.
  • except: If an error occurs inside the try block, Python immediately jumps here. Instead of crashing, you can log the error and return a safe message to the user.
  • finally: This code runs no matter what. Even if an error happens, even if the function returns early. In backend development, this is absolutely crucial for databases. If your database query crashes, the finally block ensures that the database connection is safely closed, preventing server resource leaks.
