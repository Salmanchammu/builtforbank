Chapter 1: The Foundations of the Web (HTML & CSS)

1.1 Introduction to the Web: Clients and Servers

Before we look at a single line of code, you must understand how the internet actually works. When you open a browser like Google Chrome or Microsoft Edge, your computer acts as a Client. When you type in a URL (like www.smartbank.com), your browser doesn't magically know what to display. Instead, it sends an HTTP Request over the internet.

This request travels through miles of fiber optic cables until it reaches another computer, usually sitting in a massive, air-conditioned warehouse. This computer is called a Server. Why? Because its only job in life is to serve data to clients. In the SmartBank project, your Python code (using Flask) is the Server, and the browser running your HTML/JS is the Client.

The Server receives the request, processes it, and sends back an HTTP Response. This response contains three main ingredients: HTML (the structure), CSS (the paint), and JavaScript (the muscle). Let's dive into the first two.

1.2 HTML: The Skeleton of the Web

HTML stands for HyperText Markup Language. Notice that it is not a programming language. You cannot write logic in HTML. You cannot say, "If the user is logged in, show this button." HTML is purely a markup language used to define the structure of a document.

Tags and Elements

HTML is written using tags. Think of tags like containers. You open a container, put something inside, and then close it.

<h1>Welcome to SmartBank</h1>
<p>Your money is safe with us.</p>

In the example above, <h1> is the opening tag, and </h1> is the closing tag. Everything together (the tags and the text inside) is called an HTML Element.

The DOM (Document Object Model)

This is a crucial concept for your viva. When the browser downloads your HTML file, it doesn't just read it like a book. It converts it into a giant family tree called the Document Object Model (DOM).

Imagine a real tree. The root is the <html> tag. The branches are the <head> and <body> tags. The leaves are the text inside your paragraphs. We care about the DOM because later, we use JavaScript to climb this tree and change the leaves without reloading the page!

Semantic HTML

In the old days, developers used <div> tags for absolutely everything. A header was a div, a footer was a div, a sidebar was a div. This is terrible for two reasons:

  1. Accessibility: Blind users use "Screen Readers" to browse the web. If everything is a <div>, the screen reader doesn't know where the main content starts and where the navigation menu ends.
  2. SEO (Search Engine Optimization): Google's robots scan websites to understand them. They look for meaning.

In SmartBank, we use Semantic HTML. Instead of <div id="nav">, we use the actual <nav> tag. We use <header>, <main>, <section>, and <footer>. This gives our HTML structural meaning.

1.3 CSS: The Paint and the Layout

CSS stands for Cascading Style Sheets. If HTML is the skeleton, CSS is the skin, clothes, and makeup. Without CSS, every website would look like a boring Word document from 1995.

Selectors and Specificity

To style an HTML element, you have to "select" it. There are three main ways to do this:

  • Tag Selector: p { color: red; }. This selects every single paragraph on the page. It's very broad.
  • Class Selector: .btn { color: blue; }. Notice the dot (.). A class is a reusable label you can stick on multiple elements. You might have 20 buttons on your page, and they all use the class="btn".
  • ID Selector: #header { background: black; }. Notice the hashtag (#). An ID is a unique serial number. You can only use it once per page.

If you give an element both a class that says "be blue" and an ID that says "be red", the ID wins. This is called Specificity. IDs are more specific than classes, so they override them.

The Box Model

If you learn nothing else about CSS, learn this: Every single element on a webpage is a rectangular box. Even if a button looks round because you gave it curved corners, the browser calculates it as a box. This is the Box Model.

The Box Model consists of four layers, starting from the inside out:

  1. Content: The actual text or image (e.g., the word "Submit").
  2. Padding: The invisible cushioning inside the box. If you add padding, the distance between the text and the border increases.
  3. Border: The visible line that wraps around the padding.
  4. Margin: The invisible forcefield outside the box. If you add margin, it pushes other boxes away from this one.

Flexbox vs. CSS Grid

In SmartBank, you used modern CSS layout tools to position things on the screen.

Flexbox (display: flex): Imagine you have 5 buttons and you want them to sit in a perfectly straight line, evenly spaced out. Flexbox is designed for 1-Dimensional layouts (either a row OR a column). It makes elements "flexible" so they stretch or shrink to fill the available space.

CSS Grid (display: grid): Imagine a chessboard. You have both rows AND columns. Grid is designed for 2-Dimensional layouts. In your SmartBank dashboard, you have a grid of cards (Total Balance, Active Loans, etc.). Grid allows you to define exactly how many columns you want (e.g., grid-template-columns: 1fr 1fr 1fr creates 3 equal columns).

Responsive Design and Media Queries

How does SmartBank look good on both a massive 32-inch monitor and a tiny 5-inch iPhone? Through Media Queries.

@media (max-width: 768px) {
    .dashboard-grid {
        grid-template-columns: 1fr;
    }
}

This code tells the browser: "If the screen width ever drops below 768 pixels, change the dashboard grid to have only 1 column." This forces the cards to stack vertically on top of each other, making them easy to scroll through on a phone.
