Chapter 2: JavaScript (The Muscle and Behavior)

2.1 What is JavaScript?

If HTML is the skeleton and CSS is the paint, JavaScript (JS) is the muscle. It is the programming language of the browser. Without JS, a webpage is just a static poster. You can look at it, but it doesn't do anything. With JS, you can click buttons, animate charts, and send money across the internet without ever refreshing the page.

JavaScript runs client-side. This means the code is executed directly by the user's computer (inside Chrome, Safari, etc.), utilizing their CPU and RAM, rather than the server's resources.

2.2 Variables: The Memory Boxes

A variable is simply a named box in the computer's memory where you can store data. Over the years, JavaScript has changed how we create these boxes. You must know the difference for your viva.

  • var (The Old Way): Before 2015, this was the only way to declare variables. The problem with var is that it "leaks". If you create a var inside an if statement, it can accidentally be accessed outside that if statement, causing massive, confusing bugs. Professional developers avoid var today.
  • let (The Flexible Way): Introduced to fix var. A let variable is block-scoped. This means it only exists inside the curly braces {} where it was created. You can change (reassign) the value of a let variable at any time.
  • const (The Strict Way): Short for constant. Like let, it is block-scoped. However, once you assign a value to a const, you can never reassign it. You should use const by default for everything, only switching to let if you know the value needs to change later (like a counter).

2.3 Data Types and Structures

JavaScript needs to know what kind of data it's holding in those variables.

  • Strings: Text. Wrapped in quotes. "SmartBank"
  • Numbers: Integers and decimals. 42 or 3.14
  • Booleans: True or False. Used for logic checks. isLoggedIn = true
  • Arrays: A list of items, ordered by an index starting at 0. const users = ["Salman", "Admin"]. To get "Salman", you ask for users[0].
  • Objects: The most important structure in JS. An object is a collection of Key-Value pairs, wrapped in curly braces. Sound familiar? It looks exactly like a Python Dictionary!
    const user = { name: "Salman", role: "Admin" };

2.4 Functions and Scope

A function is a reusable block of code designed to perform a single task. You write it once, and you can "call" it as many times as you want.

function calculateInterest(principal, rate) {
    return principal * rate;
}

Scope refers to where a variable is visible. If you create a variable inside a function, it is trapped in that function. The rest of your code has no idea it exists. This is called Local Scope, and it prevents variables from colliding with each other.

2.5 The DOM and Event Listeners

We mentioned the DOM (Document Object Model) in Chapter 1. It is the tree structure of your HTML. JavaScript's superpower is its ability to manipulate this tree in real-time.

Selecting Elements

To change an HTML element, JS must first find it. We use methods like:

  • document.getElementById('balanceDisplay'): Finds the one specific element with that ID.
  • document.querySelector('.card'): Finds the first element that has the class "card".

Changing Elements

Once you grab an element, you can modify it using .innerHTML or .textContent. For example, if you fetch a user's balance from the server, you can instantly display it by writing:
document.getElementById('balance').textContent = '₹5,000';

Event Listeners

A webpage is useless if it doesn't respond to the user. We use Event Listeners to "listen" for user actions (clicks, typing, scrolling).

const button = document.getElementById('submitBtn');
button.addEventListener('click', function() {
    alert('Money transferred successfully!');
});

2.6 Asynchronous JavaScript (Crucial for Viva)

This is arguably the most important JavaScript concept you need to know for a full-stack project like SmartBank.

JavaScript is Single-Threaded. Think of a restaurant with only one waiter. That waiter has to take orders, bring food, and collect bills. If the waiter goes to the kitchen to wait for a steak to cook, no other tables get served. The entire restaurant freezes.

In code, if JavaScript asks the Python server for a list of 10,000 transactions, it takes time over the internet. If JS simply waited, the browser would completely freeze. The user couldn't scroll, couldn't click, couldn't do anything.

Promises and Async/Await

To solve this, JS uses Asynchronous code. We use a feature called a Promise. A Promise is exactly what it sounds like: JS says, "I promise to go get this data, but I'll let you know when I'm done."

In modern code, we write this using the async and await keywords.

async function getUserData() {
    console.log("Fetching data...");
    const response = await fetch('/api/user/profile');
    const data = await response.json();
    console.log("Data received!", data);
}

When JS hits the await keyword, it effectively tells the browser: "Send this network request, but don't freeze. Go ahead and let the user keep clicking and scrolling. Whenever the server finally replies, come exactly back to this line of code and continue." This creates a smooth, lag-free user experience.

JSON (JavaScript Object Notation)

You'll notice response.json() in the code above. JSON is the universal language of the internet. Python and JavaScript are completely different languages. They cannot read each other's code. But they both understand text.

JSON takes a JavaScript Object or a Python Dictionary and converts it into pure, plain text. The Python server sends this text over the internet. When it arrives in the browser, JS parses the text back into a usable JavaScript Object.
